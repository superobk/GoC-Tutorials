# GoC 魔法画笔打字训练营：设计规格

## 目标

在现有课程首页新增一块儿童打字练习区，让孩子通过短回合“画笔任务”掌握 GoC 常用指令。训练与现有真实 GoC 运行器共用安全的绘图能力，不引入账号、网络请求或后端。

## 用户体验

训练营有两档，由孩子点击切换：

1. **热身档：指令名**
   - 屏幕给出一个画笔任务和一张指令卡，例如“向前走一小步”与 `fd`。
   - 孩子只需输入指令名（如 `fd`）并按 Enter 或点击提交。
2. **挑战档：完整命令**
   - 屏幕给出任务、目标命令和参数语义，例如“向前走 100 格”。
   - 孩子输入完整 GoC 命令（如 `fd(100);`）。

每一回合聚焦一条常用指令，持续 20–30 秒。答对会播放既有运行器绘制出的对应动作；拼写不匹配时，输入框与提示文本展示第一个错误或缺失字符，孩子可立即重试，不因单题错误中断游戏。

## 题库与范围

首版使用固定、可轮换的常用绘图指令：`pen`、`fd`、`bk`、`rt`、`lt`、`c`、`up`、`down`、`moveTo`、`lineTo`、`o`、`r` 与 `for`。

挑战档只校验该关要求的完整命令；参数会是明确的常数，不要求变量、输入或多行代码。`for` 关是下文说明的唯一循环例外。这样训练目标始终是准确输入命令，而非解算法题。

`for` 是唯一的两行挑战题：热身档输入 `for`；挑战档输入固定次数循环的首行 `for(i=0;i<4;i++)`，并由题目提供循环体 `fd(60); rt(90);`。答对后运行器执行完整的正方形代码，让孩子看到“重复写一次，画笔走四次”的效果。

## 界面组件与状态

- 训练营入口区：标题、档位按钮、任务卡、输入框与提交按钮。
- 画布预览：复用现有 canvas 渲染管线，在正确提交后展示一次命令效果。
- 进度面板：当前连击、正确题数、正确率和本机最高连击。
- 提示区：未输入、拼写错误、完成回合与下一关引导。
- 本地存储：仅保存最高连击、总答题数和正确题数；没有姓名、账号或上传行为。

## 判定与容错

- 忽略输入前后空格；挑战档保留命令拼写、括号、参数和分号的训练要求。
- 大小写按 GoC 指令规则校验；`moveto` 不能代替 `moveTo`。
- 错误提示只显示首个不匹配位置或“还缺少 …”，不直接泄露整条答案。
- 正确后禁用当前提交，短暂显示绘制结果，再切至下一张任务卡；孩子可通过“下一关”提前切换。
- 安全运行器报错时，显示友好提示并保持游戏可继续，不执行任意 JavaScript。

## 实现边界

- 新增一个独立的训练数据与判定模块，避免把题库逻辑堆进页面组件。
- 只在 `app/page.tsx`、样式、训练模块和测试文件中进行必要修改。
- 继续使用已有的 `goc-runner.mjs` 输出绘图操作；不使用 `eval`。
- 不改动既有“真实运行”、课程卡或决赛题练习的行为。

## 验收与测试

1. 单元测试覆盖：热身档精确匹配、前后空格容错、挑战档必须包含正确标点、首个错误位置提示、统计更新。
2. 页面渲染测试确认训练营标题、两个档位与输入控件可见。
3. 执行 `npm test` 与 `npm run lint`；两项均应通过。
4. 手动核验：完整命令答对后，画布出现对应的绘制效果，且刷新页面仍保留最高连击与统计。
